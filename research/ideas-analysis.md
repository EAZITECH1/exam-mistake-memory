# Idea analysis

Four candidates from my own daily workflows, scored against the three judging criteria: real problem, well-crafted memory rules, provable writes.

## 1. Exam Mistake Memory — chosen

A study agent that stores every practice-question mistake and drills weaknesses across sessions.

Why it won: the pain is mine and it's daily. I'm a Pharm.D student at LASUCOM grinding past questions; my AI study sessions restart from zero every single time. Mistakes are the highest-value data in exam prep and they evaporate when the session closes. Blob generation is organic (every wrong answer writes), the behaviour change between sessions is easy to show on camera, and nobody else in a Web3 hackathon submits a pharmacy study tool. Generalizes to any exam with a config block, so it fits a prompt library.

## 2. Client style-memory agent

Per-client namespaces holding editorial decisions, tone rules, and my banned-words list for Web3 writing work (SUI recaps, 0G content).

Why it lost: too close to the translator example in the hackathon brief. Solving the example provided is the opposite of "the prompt we haven't thought of."

## 3. ARG puzzle memory

Logs every attempted decode and failed hypothesis per puzzle namespace so the agent never re-suggests a dead end (Caesar shifts already tried, rune mappings ruled out).

Why it lost: genuinely weird, which judges like, but blob volume is thin and demoing it in under 3 minutes means explaining the puzzle before explaining the prompt.

## 4. Copytrading research memory

Stores wallet and bot evaluations, rug flags, and verdicts; recalls before evaluating anything "new."

Why it lost: crypto research agents will be the most crowded lane in this hackathon.
